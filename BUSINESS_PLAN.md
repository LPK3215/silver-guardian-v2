# 银发守护 —— 业务规划文档

> 基于 Yuxi (v0.7.1.beta1) 改造的面向养老场景 AI 对话与知识库问答系统
> 核心思路：**先直接接入跑起来，再拓展独特业务**

---

## 一、Yuxi 天然支持什么

以下是 Yuxi 开箱即用的能力，**不需要写任何代码**，全部通过 Web 管理界面配置：

### 1.1 对话智能体（零代码配置）

| 能力 | 说明 | 银发守护怎么用 |
|------|------|---------------|
| 智能体管理 | 界面创建 Agent，配置名称、头像、描述 | 创建多个养老场景 Agent |
| 系统提示词 | 每个 Agent 独立配置 system_prompt | 定义 Agent 角色和语气 |
| 模型选择 | 界面切换每个 Agent 用什么模型 | 统一用 DeepSeek |
| 运行配置 | 最大步数、摘要阈值、保留消息数等 | 按场景调优 |
| 多 Agent 切换 | 聊天界面快速切换不同 Agent | 用户选健康顾问/政策助手/陪伴 |

### 1.2 知识库 RAG（需完整模式，非 LITE）

| 能力 | 说明 | 银发守护怎么用 |
|------|------|---------------|
| 文档上传 | PDF/Word/PPT/Excel/图片/纯文本 | 上传养老政策、健康指南等文档 |
| 自动解析 | MinerU/PaddleX/RapidOCR 解析为 Markdown | 政策 PDF 自动提取文本 |
| 自动分块 | 按策略切分 chunk | 文档拆成可检索片段 |
| 向量检索 | Milvus 语义搜索 | 用户提问自动检索相关文档 |
| 引用来源 | 回答附带文档出处 | 回答政策问题时标注来源 |
| 知识导图 | 自动生成知识库结构树 | 快速了解知识库覆盖范围 |
| 检索测试 | 界面直接测试检索效果 | 验证知识库质量 |
| 权限控制 | 全局/部门/指定人三级权限 | 不同机构隔离知识库 |

> **关键注意**：知识库依赖 Milvus，LITE 模式会跳过知识库接口。要用知识库，不能用 `LITE_MODE=true`，需要至少启动 milvus + etcd + minio。

### 1.3 多智能体编排

| 能力 | 说明 | 银发守护怎么用 |
|------|------|---------------|
| 子智能体 | 创建 SubAgent，被主 Agent 调用 | 主 Agent 分发到专科子 Agent |
| task 工具 | 主 Agent 同步调用子 Agent | 简单任务直接拿结果 |
| 异步子智能体 | start/status/events/cancel/await | 长任务并行处理 |
| 文件共享 | 子 Agent 可读父会话上传文件 | 子 Agent 分析用户上传的体检报告 |

### 1.4 内置工具（界面勾选即可启用）

| 工具 | 说明 | 银发守护怎么用 |
|------|------|---------------|
| `tavily_search` | 网页搜索 | 搜索最新养老政策新闻 |
| `ask_user_question` | 向用户发起交互式提问 | 追问症状细节 |
| `present_artifacts` | 展示产物文件 | 展示生成的护理计划文档 |
| MySQL 查询 | 只读 SQL 查询 | 查询机构老人信息（可选） |
| 知识库工具 | list/query/find/open 文档 | Agent 自动检索知识库 |

### 1.5 Skills 技能系统

| 能力 | 说明 | 银发守护怎么用 |
|------|------|---------------|
| 推荐 Skills | 界面一键安装 | 安装文档处理等通用技能 |
| ZIP 上传 | 打包上传自定义 Skill | 封装养老专属提示词模板 |
| 远程安装 | 从 GitHub/ModelScope 拉取 | 获取社区技能 |
| 在线编辑 | 网页直接编辑 SKILL.md | 迭代优化技能提示词 |
| 依赖管理 | Skill 间可声明依赖 | 组合多技能 |

### 1.6 MCP 集成

| 能力 | 说明 | 银发守护怎么用 |
|------|------|---------------|
| 远程 MCP | Streamable HTTP / SSE | 接入外部养老数据 API |
| 本地 MCP | Stdio 进程 | 接入本地数据库/文件系统 |
| 工具粒度控制 | 禁用特定 MCP 工具 | 精细化权限管理 |

### 1.7 其他开箱即用能力

| 能力 | 说明 |
|------|------|
| 沙盒代码执行 | Agent 可写 Python 代码并运行 |
| 文件系统 | 上传/下载/预览文件 |
| 多租户 | 部门/用户/权限管理 |
| 模型管理 | 界面配置多供应商（DeepSeek/OpenAI 等） |
| 品牌定制 | YAML 配置改名/Logo/主题色 |
| 仪表盘 | 使用统计概览 |
| 后台任务 | 异步任务查询管理 |

## 二、银发守护能直接接入什么

### 2.1 零代码直接接入（界面配置即可）

```
用户提问 → DeepSeek 模型 → 根据系统提示词回答
```

**第一步：配模型**
- 系统设置 → 模型配置 → 启用 DeepSeek 供应商
- 填入 `DEEPSEEK_API_KEY`
- 添加 `deepseek-chat` 模型

**第二步：建智能体（纯配置，不写代码）**

| Agent 名称 | system_prompt 核心方向 | 关联知识库 |
|-----------|----------------------|-----------|
| 银发健康顾问 | 你是专业的老年健康顾问，用通俗语言解答健康问题，强调非诊断性建议 | 健康指南库 |
| 养老政策助手 | 你是养老政策专家，准确引用政策原文，注明来源和适用条件 | 政策法规库 |
| 生活陪伴 | 你是温暖的陪伴者，用亲切语气聊天，鼓励积极生活态度 | 生活指南库 |
| 护理知识助手 | 你是专业护理顾问，面向护工提供规范操作指导和应急处理流程 | 护理规范库 |

**第三步：传文档建知识库**
- 知识库页面 → 新建知识库 → 上传养老相关文档
- 系统自动解析、分块、建索引
- Agent 配置中关联知识库

**以上三步完成后，银发守护就能跑了。不需要写一行代码。**

### 2.2 需要注意的关键决策

#### 决策一：LITE 模式 vs 完整模式

| 模式 | 启动服务 | 有知识库 | 有知识图谱 | 资源占用 |
|------|---------|---------|-----------|---------|
| LITE | postgres + redis + minio + api + web | ❌ | ❌ | 低 |
| 完整 | 上述 + milvus + etcd + neo4j | ✅ | ✅ | 高 |

**建议**：银发守护的核心价值是知识库问答，应该用**完整模式**。如果暂时不需要知识图谱，可以不启动 neo4j，但 milvus + etcd 必须有。

启动命令（完整模式但跳过图谱）：
```powershell
docker compose up -d postgres redis minio milvus etcd api web
```

#### 决策二：模型选择

DeepSeek 是 OpenAI 兼容协议，Yuxi 内置了 DeepSeek 供应商模板，界面直接启用即可。

| 模型 | 用途 |
|------|------|
| `deepseek-chat` | 对话主力模型 |
| `deepseek-reasoner` | 复杂推理（可选） |

## 三、直接接入后的操作步骤

### 3.1 启动准备

```powershell
cd d:\Develop\SourceCode\agent-projects\silver-guardian-yuxi\Yuxi

# 1. 编辑 .env，填入真实的 DEEPSEEK_API_KEY
# 2. 拉取 Docker 镜像
.\scripts\init.ps1

# 3. 启动服务（完整模式，含知识库）
docker compose up -d postgres redis minio milvus etcd api web
```

### 3.2 界面配置（全部在 http://localhost:5173 操作）

```
1. 首次访问 → 设置管理员账号密码
2. 系统设置 → 模型配置 → 启用 DeepSeek → 填 API Key → 添加 deepseek-chat
3. 知识库 → 新建知识库 → 上传养老文档
4. 智能体管理 → 新建智能体 → 写 system_prompt → 关联知识库
5. 开始对话
```

### 3.3 品牌定制（已预配置）

已创建 `info.local.yaml`，界面会显示"银发守护"。如需更换 Logo 和主题色：

- Logo：替换 `web/public/favicon.svg` 和 `web/public/avatar.jpg`
- 主题色：编辑 `web/src/assets/css/base.css` 中的 `--main-*` 变量
- 同步修改 `web/src/stores/theme.js` 中的 `colorPrimary`

## 四、后续拓展方向（需要写代码）

当基础业务跑通后，以下拓展需要开发工作：

### 4.1 低成本拓展（改配置/提示词即可）

| 拓展 | 方式 | 难度 |
|------|------|------|
| 更多专科 Agent | 界面创建新 Agent + 写提示词 | ⭐ |
| 知识库分类细化 | 建多个知识库，按主题分 | ⭐ |
| 网页搜索能力 | 配置 `TAVILY_API_KEY`，启用 tavily_search 工具 | ⭐ |
| 子智能体分工 | 创建 SubAgent，主 Agent 调用 | ⭐⭐ |
| 自定义 Skills | 写 SKILL.md 封装养老提示词模板 | ⭐⭐ |
| MCP 接入外部数据 | 配置 MCP 服务连养老数据 API | ⭐⭐ |

### 4.2 中等成本拓展（需要写后端代码）

| 拓展 | 方式 | 难度 |
|------|------|------|
| 自定义工具 | `@tool` 装饰器注册（如查药品库、查社区服务） | ⭐⭐⭐ |
| 自定义 Agent | 继承 BaseAgent，自定义 Context 和 Graph | ⭐⭐⭐ |
| 健康数据查询 | MCP 服务对接 IoT 设备数据 | ⭐⭐⭐ |
| 用药提醒 | 自定义工具 + 定时任务 | ⭐⭐⭐ |

### 4.3 高成本拓展（需要前后端改造）

| 拓展 | 方式 | 难度 |
|------|------|------|
| 语音交互 | 接入 STT/TTS，前端录音播放 | ⭐⭐⭐⭐ |
| 适老化大字界面 | 前端 CSS 改造 | ⭐⭐⭐⭐ |
| 家属端联动 | 多租户权限 + 消息推送 | ⭐⭐⭐⭐ |
| 紧急求助识别 | Agent 中间件，语义识别紧急情况 | ⭐⭐⭐⭐ |
| 微信小程序 | 独立前端，调 Yuxi API | ⭐⭐⭐⭐⭐ |

## 七、总结

**银发守护的业务可以 100% 直接介入 Yuxi 的天然能力**：

1. **对话** → Yuxi 的智能体对话界面，配 DeepSeek 即用
2. **知识库** → Yuxi 的 RAG 知识库，上传文档即用
3. **多角色** → Yuxi 的多 Agent + 子智能体，界面配置即用
4. **管理** → Yuxi 的多租户 + 权限系统，开箱即用

**先跑起来**：配模型 → 建 Agent → 传文档 → 开始对话
**再拓展**：自定义工具 → 自定义 Agent → 前端改造 → 语音/小程序
