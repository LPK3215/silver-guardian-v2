# 安全策略

## 报告漏洞

如果你发现安全漏洞，请**不要**通过公开 Issue 报告。

请发送邮件至 `17538203215@163.com`，并在邮件中包含：

- 漏洞的详细描述
- 复现步骤
- 影响范围评估
- 建议的修复方案（如有）

我们会在收到报告后 48 小时内确认，并在修复后发布安全公告。

## 安全配置建议

### 生产环境部署

1. **修改默认密码**：首次部署后立即修改管理员密码和数据库密码
2. **配置 CORS**：设置 `YUXI_CORS_ORIGINS` 为你的前端域名，不要使用 `*`
3. **设置 JWT Secret**：`JWT_SECRET_KEY` 必须使用随机生成的强密钥
4. **限制数据库端口**：PostgreSQL、Redis、MinIO 端口不应对外暴露
5. **使用 HTTPS**：通过 Nginx 或反向代理启用 TLS

### API Key 安全

- API Key 绑定到具体用户，不支持匿名使用
- 定期轮换 API Key
- 不要在代码仓库中提交 `.env` 文件（已在 `.gitignore` 中排除）

### 沙盒安全

- 沙盒用于执行智能体工具代码，默认有超时和输出大小限制
- 生产环境建议使用 Kubernetes 后端而非 Docker 后端
- 沙盒镜像应定期更新
