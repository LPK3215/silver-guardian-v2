社会服务赛道 社会公益 智慧助老

创意介绍

银发守护是一个面向养老场景的 AI 知识库与智能体平台。用户通过统一的 Web 界面与 AI 助手对话，系统自动从文档知识库、结构化数据库、知识图谱等多源知识中检索信息，并给出带来源引用的回答。管理员可在同一平台上管理知识文档、维护数据表、查看对话历史与运营指标。

核心亮点  LangGraph Agent 工作流支持多步骤推理和工具调用；RAG 双层检索（文档 + 数据库）+ 知识图谱实体关联，让 AI 回答不局限于关键词匹配，而是具备真正的语义理解和溯源能力。

想解决的问题

📚

信息孤岛

养老政策文件在民政、卫健、人社各条线分散发布，老人档案和健康记录散落在不同的 Excel 表格里，知识无法统一检索。

⏳

知识更新滞后

政策文件更新频繁，传统系统每次改内容都要找开发人员改代码，管理人员无法自主维护知识库。

⚠️

AI 回答不可信

传统聊天机器人的回答是"黑盒"，用户无法判断信息来源，不敢信任。必须有来源引用和原文预览。

🧠

复杂场景无法应对

养老场景常需要多步推理——例如"某老人血压偏高，根据社区政策能申请什么服务"，需要同时查健康数据、政策文档和服务清单。

核心功能

  • 1
    LangGraph Agent 智能对话 基于 LangGraph v1 的 Agent 工作流系统，支持流式对话（SSE）、多步骤推理、工具调用和子 Agent 协作，可编排复杂养老场景的对话链路。
  • 2
    RAG 多源知识检索 + 来源溯源 文档知识（Milvus 向量检索）+ 结构化数据库（PostgreSQL SQL 检索）双层 RAG，回答时标注来源并可点击预览原文，支持 Markdown、PDF、Word 等多种文档格式。
  • 3
    LightRAG 知识图谱 基于 LightRAG + Neo4j 的知识图谱引擎，自动抽取实体关系，支持图谱可视化和图谱增强问答，让 AI 理解"张三属于哪个社区、该社区有什么政策"这类实体关联问题。
  • 4
    知识库管理后台 Markdown 文档上传/编辑/全文检索 + 结构化数据表增删改查 + 预览弹窗 + 系统运营仪表盘。管理员可自主维护知识库，无需开发人员介入。
  • 5
    文档智能解析 集成 Docling 和 RapidOCR，支持 PDF、Word、图片等多种格式的自动解析入库，降低知识录入门槛。
  • 6
    沙箱安全执行 Agent 可在隔离的 Docker 沙箱中安全执行代码，支持数据分析、报表生成等需要计算的场景，不影响主系统安全。

价值与意义

社会价值：面向真实养老场景，用 AI 降低老人获取陪伴、健康管理和公共服务信息的门槛。知识图谱让 AI 回答具备实体关联和语义理解能力，而非简单的关键词匹配。

产品价值：不是概念稿，是已经具备完整前后端、Agent 工作流、知识图谱、文档解析的全功能可运行 Demo，支持 Docker Compose 一键部署。

技术价值：LangGraph Agent 多步骤推理 + RAG 多源检索 + LightRAG 知识图谱 + 沙箱执行的四合一架构，在养老 AI 领域具有创新性。来源透明设计让 AI 回答可信可溯源。
